# SparesX Project Context

Project Name: SparesX  
Tech Stack:

- Next.js (App Router)
- MongoDB (Mongoose)
- Node.js
- Tailwind CSS
- JWT authentication

Project Purpose:
A marketplace where mobile technicians can list and sell spare parts.

Core Modules:

- User authentication (Technician, Admin)
- Product listing by technicians
- Admin approval workflow
- Product search & filters
- Order management
- Dashboard (Admin + Technician)

Coding Rules:

- Use Next.js App Router
- Prefer Server Components where possible
- API routes in /app/api
- Use async/await only
- MongoDB via Mongoose models
- No deprecated Next.js APIs
- Clean, readable, production-level code

Output Expectations:

- Always explain complex logic briefly in comments
- Avoid unnecessary libraries
- Follow existing folder structure
